Results

Overview

This Results page brings together the outcomes of every method used in the project. The table below summarizes what that technique revealed about Disney park wait-time behavior and, when applicable, how well it performed on the busy vs. not_busy prediction task.

Method Key Result(s) Performance Metric(s) Takeaway / Efficacy
K-Means Clustering Best k = 2 by silhouette; two clear day-types Silhouette peak at k=2 (best separation) Strong, simple structure: days split into “typical” vs “peak-crowd.”
Hierarchical Clustering Best k = 3 by silhouette (~0.61); three crowd tiers Silhouette max at k=3 (~0.61) More granular than k-means: quiet / moderate / peak days.
Principal Component Analysis (PCA) PC1 explains 83.5% variance; PC2 adds 12.2% Total variance in first 2 PCs ≈ 95.7% Wait patterns are dominated by one “crowd intensity” axis plus a smaller secondary factor.
Decision Tree High test performance; clear class splits in tree Confusion matrix + feature importance Interpretable baseline; key splits align with wait-time intensity.
Naive Bayes (MultinomialNB) Perfect test classification Accuracy = 1.00;
Confusion matrix = [[3,0],[0,7]];
Precision/Recall/F1 = 1.00 for both classes 		Very strong on this dataset; daily wait summaries contain highly separable signal.
AdaBoost Best settings: depth=2, n_estimators=10, learning_rate=1.0 Accuracy = 0.9;
Confusion matrix = [[2,1],[0,7]]; 		Boosting captures crowd signal quickly; medium-strength weak learners worked best.
